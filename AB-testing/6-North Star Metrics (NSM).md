# 🧠 1. North Star Metric (NSM)

---

## 🎯 What it is

> A **single metric** that best captures the **long-term value delivered to users**

---

## 🔥 Properties of a Good NSM

| Property     | Meaning                                 |
| ------------ | --------------------------------------- |
| User-centric | Reflects user value (not just business) |
| Long-term    | Not easily gamed                        |
| Sensitive    | Moves with product changes              |
| Actionable   | Teams can influence it                  |

---

## 📊 Examples

| Product       | NSM                      |
| ------------- | ------------------------ |
| Social Media  | Daily Active Users (DAU) |
| YouTube-like  | Watch Time               |
| E-commerce    | Orders per user          |
| Uber-like     | Trips per user           |
| Search Engine | Successful searches      |

---

## 🚨 Interview Trap

### ❌ Bad NSM:

* Clicks
* Page views

👉 Easily gamed (clickbait, spam)

---

## ✅ Strong Answer Pattern

> “I’d define a North Star metric that captures long-term user value, such as X, and ensure it aligns with both engagement and retention.”

# 🔁 2. User Retention (CRITICAL for FAANG)

---

## 🧠 What it is

> Probability that a user **comes back over time**

---

## 📊 Types of Retention

| Type              | Definition                 |
| ----------------- | -------------------------- |
| Day N Retention   | % users returning on day N |
| Rolling Retention | Returned on or after day N |
| Cohort Retention  | Track users by signup date |

## 🔥 Key Insight

* **Good product → retention curve flattens**
* **Bad product → goes to zero**

---

## 🎯 Why Retention Matters

* Strong predictor of **long-term growth**
* Prevents **vanity metrics**

---

## 💬 Interview Gold Lines

> “Retention is a better indicator of product-market fit than short-term engagement.”

> “I’d analyze retention across cohorts to detect long-term impact.”

---

## 🚨 Common Mistakes

* Only looking at CTR / DAU
* Ignoring churn
* Not segmenting cohorts

## 🔥 Real Examples

| System             | Trade-off                         |
| ------------------ | --------------------------------- |
| Search ranking     | Fast results vs best ranking      |
| Ads system         | Real-time bidding vs optimal ad   |
| Recommender system | Instant suggestions vs best match |
| Fraud detection    | Speed vs correctness              |

---

## 🧠 Key Decisions

### 1. When to prioritize LATENCY

* Real-time systems (search, ads)
* User-facing interactions

---

### 2. When to prioritize ACCURACY

* Offline batch systems
* Critical decisions (fraud, medical)

---

## ⚙️ Techniques to Balance

| Method             | Idea                           |
| ------------------ | ------------------------------ |
| Caching            | Precompute results             |
| Two-stage models   | Fast filter → slow rerank      |
| Approximation      | Slight accuracy loss for speed |
| Model distillation | Smaller faster model           |
| Feature pruning    | Reduce compute                 |

---

## 💬 Interview Gold Answer

> “I’d design a two-stage system where a lightweight model retrieves candidates quickly, followed by a more complex model for re-ranking to balance latency and accuracy.”
